# 13. Автотесты

## SQL (`ClayCompositeSqlBuilder` + `ClayDataQuery.BuildSingleClause`)
- Группа И/ИЛИ с вложенностью → корректный `WHERE` со скобками.
- Лист с неизвестной колонкой → отбрасывается; SQL-текст в `Column` не попадает в запрос.
- Значения параметризуются; SQL в значении не меняет текст запроса.
- Повтор колонки → уникальные имена параметров.
- Типы (03): `IsNull/IsNotNull`→`IS NULL/IS NOT NULL`; Date/Decimal параметризуются; парсинг инвариантный.

## Модель
- `ClayFilterGroupNode.Clone()` — глубокая рекурсивная копия (правка копии не трогает оригинал).
- `ColumnFilter.Clone()` — оба условия и `Source` скопированы.
- `ColumnTypeRegistry.FromClr` — корректный дескриптор по CLR-типу (вкл. Nullable).

## Диалог/грид
- «Сбросить» → пустой корень, фильтрация снята во всех путях загрузки.
- «Применить» → грид получает копию дерева (не ссылку на черновик).
- Маршрут клика: `Source=ColumnDialog` → колоночный диалог; иначе — настраиваемый.
- Оператор без значения обнуляет значение листа.

## Описание/индикатор
- Построитель: скобки, И/ИЛИ, подписи операторов, формат даты.
- Счётчик активных условий считает листья рекурсивно (вкл. второе условие).

## Сериализация (если делалась 12)
- JSON round-trip полиморфного дерева; `ParamName` не сериализуется.
