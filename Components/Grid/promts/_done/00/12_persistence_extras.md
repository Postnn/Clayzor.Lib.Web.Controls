# 12. Сериализация / URL / индикатор / локализация (опционально)

## JSON
- `ClayFilterJsonConverter : JsonConverter<IClayFilterNode>` с дискриминатором `$type`:
  `"group"`→`ClayFilterGroupNode`, `"column"`→`ColumnFilter`.
- Группа: `Logic`, `Nodes`. Лист: `Column`, `Operator`, `Value`, второе условие,
  `Source`. `ParamName`/`SecondParamName` **не** сериализовать (переназначаются при сборке SQL).
- `object? Value` — с типом (через `JsonElement`); десериализация даёт `Clone`-эквивалент.
  Для восстановления типа значения использовать `ColumnTypeRegistry.FromKind`/дескриптор.

## URL
- `ClayFilterUrlHelper`: дерево → JSON → сжатие → Base64Url в query и обратно;
  при наличии параметра грид инициализирует `_filterRoot` и грузит отфильтрованным.

## Индикатор активного фильтра
- Счётчик условий — рекурсивно по листьям (с учётом второго условия листа);
  на кнопке открытия; при нуле скрывать.

## Локализация
- Подписи операторов и И/ИЛИ — из `GetOperatorLabel`/построителя описания;
  строки кнопок («добавить условие/группу», «удалить», «Применить», «Сбросить», «Отмена»)
  — один источник, без хардкода в разметке.

## Критерии
- [ ] JSON round-trip полиморфного дерева; `ParamName` не сериализуется.
- [ ] Индикатор корректно считает листья.
- [ ] Предзагрузка из URL восстанавливает дерево.
