# QS2 — Схема dev/test-стенда

> Требует выполненного `QS1`. Правила — в `QS0`.

## Цель

Чтобы фичу можно было проверить локально, колонка `УчаствуетВБыстромПоиске` должна появиться в `scripts/dynamic-grid/schema.sql`. Это **только dev/test-стенд (LocalDB)**, не продакшн.

## Что сделать

1. В `CREATE TABLE dbo.ClayGridColumns` добавить `УчаствуетВБыстромПоиске tinyint NULL`.
   - Именно `NULL`, без `DEFAULT`. `NULL` — законное состояние и означает «не участвует» (см. QS6). `NOT NULL DEFAULT 0` не ставить: это изменило бы поведение уже существующих стендов.
2. Отдельно добавить **идемпотентный `ALTER`** для стендов, где таблица уже создана — скрипт должен быть безопасно перезапускаемым, как и остальной файл:

```sql
IF COL_LENGTH('dbo.ClayGridColumns', 'УчаствуетВБыстромПоиске') IS NULL
    ALTER TABLE dbo.ClayGridColumns ADD УчаствуетВБыстромПоиске tinyint NULL;
GO
```

3. В seed-данных грида №140 проставить осмысленные значения, чтобы было что тестировать:
   - `Название` (`Тип=2`, строка) → `1`
   - `КодИсследования` (`Тип=1`, целое) → `1`
   - `ДатаСоздания` (`Тип=3`) → на своё усмотрение, но зафиксируй решение в отчёте
   - `КодТипа` (`Тип=5`, справочник) → **`1`** — намеренно «неправильное» значение от админа, чтобы шаг QS4 было чем проверять: колонка недопустимого типа не должна попасть в поиск
   - `Активно` (`Тип=7`, логическое) → `0`
   - Seed уже обёрнут в `IF NOT EXISTS` — на стенде, где строки грида №140 уже есть, `INSERT` не сработает. Добавь рядом идемпотентный `UPDATE` значений `УчаствуетВБыстромПоиске` для `КодЗапроса = 140`, чтобы существующий стенд тоже получил тестовые данные.

4. Стиль файла соблюдать: комментарии-разделители, `GO` между батчами, кириллические имена без квадратных скобок там, где это уже так сделано.

## Чего НЕ делать

- Не менять существующие колонки, типы, ключи, триггер `TR_ClayGridUserParams_Upsert`.
- Не добавлять индексы, констрейнты `CHECK`, вычисляемые колонки.
- Не писать миграцию для продакшн-БД и не трогать ничего вне `scripts/dynamic-grid/schema.sql`.
- Не делать колонку обязательной: **вся суть задачи — приложение должно работать и без неё.**

## → verify

- Скрипт применяется на чистой БД без ошибок.
- Скрипт применяется **повторно** на уже применённой БД без ошибок (идемпотентность).
- `SELECT КодКолонки, Колонка, Тип, УчаствуетВБыстромПоиске FROM dbo.ClayGridColumns WHERE КодЗапроса = 140` возвращает 5 строк с проставленными значениями.
- Приложение на этом стенде запускается и грид №140 открывается **до** реализации остальных шагов (регресс не внесён).
