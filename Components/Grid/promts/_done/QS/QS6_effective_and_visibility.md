# QS6 — Итоговый набор колонок и видимость поля поиска

> Требует `QS3`, `QS4`, `QS5`. Правила — в `QS0`. Это «мозг» фичи — здесь сходятся все источники.

## Три источника и порядок их применения

Итоговый набор колонок быстрого поиска вычисляется строго в этом порядке:

```
1. Есть ли колонка УчаствуетВБыстромПоиске в ColumnsTable?   (QS3)
      нет → набор ПУСТ, быстрого поиска в гриде нет вообще. СТОП.
2. Админский набор  = колонки с УчаствуетВБыстромПоиске = 1   (QS3)
3. Пользовательский набор из ClayGridUserParams               (QS5)
      строка отсутствует → берём админский набор
      строка есть (в т.ч. пустая) → берём пользовательский, он ПЕРЕБИВАЕТ админский
4. Фильтр по допустимости типа                                (QS4)
      выбрасываем всё, что недопустимо по типу — независимо от того,
      кто это поставил, админ или пользователь
```

Реализовать одним методом с `///`-описанием этого порядка. Метод должен быть **чистым** (на вход — метаданные колонок и пользовательский список, на выход — набор имён колонок), чтобы его можно было покрыть тестами без БД.

### Почему пустая строка ≠ отсутствие строки

Пользователь снял все галки — это осознанный выбор. Если трактовать пустое значение как «настройки нет», админские `1` вернутся при следующей загрузке и галки самовольно восстановятся. Разделение обязательно.

## Видимость поля поиска

Поле поиска на панели управления грида показывается **тогда и только тогда**, когда итоговый набор непуст.

Скрыто, если:
- колонки `УчаствуетВБыстромПоиске` физически нет в `ColumnsTable`;
- все значения `0`/`NULL`;
- пользователь снял все галки;
- после фильтра по типу не осталось ни одной колонки (например админ поставил `1` только справочной колонке).

Отдельно: если колонки в таблице нет — быстрого поиска в гриде **нет как функционала**. Не показывать ни поле, ни чекбоксы в диалоге настройки колонок (QS8), ни пункты меню.

## Ловушка: скрытое, но активное условие поиска

Если строка поиска непуста, а набор колонок стал пустым (пользователь снял все галки), данные останутся отфильтрованными **невидимым** условием — пользователь увидит неполный список без объяснения причины.

Правило: **при опустошении набора колонок строка поиска очищается и данные перезагружаются без условия поиска.** Это часть применения настроек, а не отдельная кнопка.

## Перезагрузка данных при смене набора

Смена набора колонок поиска при непустой строке поиска меняет суть запроса. Поэтому при применении настроек:

- если строка поиска непуста и набор колонок изменился → перезагрузить данные (через тот же путь, которым грид перезагружается при смене поиска — `NotifyQueryChanged()` или его аналог, установленный в QS1);
- если строка поиска пуста → ничего не перезагружать, лишний запрос не нужен;
- сброс страницы пагинации на первую и сброс выделения при изменении поиска **уже реализованы** в гриде («Сбрасывается только при изменении сути запроса: поиск, группировка, сортировка, фильтры») — **не дублировать эту логику**, воспользоваться существующей.

## Чего НЕ делать

- Не показывать поле поиска «серым/выключенным» вместо скрытия — заказчик сказал скрывать.
- Не писать пользователю сообщений в стиле «быстрый поиск недоступен».
- Не добавлять параметр компонента для принудительного включения/выключения поля.
- Не пересчитывать набор на каждый рендер — считать при загрузке определения и при применении настроек.

## → verify

Unit-тесты (без БД, на чистом методе) — по одному на каждую строку таблицы:

| Колонка в таблице | Админ | Пользователь | Ожидаемый набор | Поле поиска |
| --- | --- | --- | --- | --- |
| нет | — | — | пусто | скрыто |
| есть | все `0`/`NULL` | нет строки | пусто | скрыто |
| есть | `Название=1` | нет строки | `Название` | видно |
| есть | `Название=1` | `''` (пусто) | пусто | скрыто |
| есть | `Название=1` | `КодИсследования` | `КодИсследования` | видно |
| есть | только `КодТипа=1` (справочник) | нет строки | пусто | скрыто |
| есть | `Название=1` | `Название,КодТипа` | `Название` | видно |
| есть | `Название=1` | `НетТакойКолонки` | пусто | скрыто |

Плюс ручная проверка: с непустой строкой поиска снять все галки → «Применить» → строка поиска очистилась, поле скрылось, данные показаны полностью.
