# 04. Вынос @code грида в ClayGrid.razor.cs (механический рефакторинг)

`ClayGrid.razor` (~80 КБ) держит всю логику в `@code` внутри разметки; отдельного
`.razor.cs` нет. Приводим к правилу «`.razor` + `.razor.cs`». Чистая механика.

## Шаги
1. Git-коммит до начала.
2. Создать `ClayGrid.razor.cs`:
   ```csharp
   public partial class ClayGrid<TEntity> : ComponentBase  // + те же интерфейсы (IClayGrid и пр.)
   {
       // сюда — ВСЁ содержимое блока @code из .razor (поля и методы) без изменений
   }
   ```
   Базовый класс/интерфейсы указать здесь (один раз). `[Parameter]`, `[Inject]`,
   `[CascadingParameter]` переносятся как есть.
3. Из `.razor` блок `@code { ... }` удалить целиком; **разметку не трогать**.
4. `dotnet build` + ручная проверка: грид ведёт себя идентично.
5. Коммит.

## Критерии
- [ ] В `.razor` нет блока `@code`.
- [ ] Поведение грида не изменилось.
- [ ] `dotnet build` без ошибок.

> Это безопасный «разогрев» перед разбиением по темам (задача 05).
