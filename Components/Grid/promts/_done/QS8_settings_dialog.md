# QS8 — Чекбокс «Быстрый поиск» в диалоге настройки колонок

> Требует `QS4`, `QS5`, `QS6`. Правила — в `QS0`.

## Цель

Пользователь меняет участие колонок в быстром поиске в `ClayColumnSettingsDialog` (открывается кнопкой `ViewColumn` на панели управления).

## Ловушка: диалог общий для трёх сценариев

`ClayColumnSettingsDialog` используется:
1. настройка колонок динамического грида — **здесь чекбокс нужен**;
2. настройка колонок статического грида — колонки `УчаствуетВБыстромПоиске` не существует, **чекбокса быть не должно**;
3. выбор колонок для печати и Excel (`ShowSorting=false`, вызывается из `ClayColumnSettingsPromptDialog`) — настройка поиска здесь бессмысленна, **чекбокса быть не должно**.

Поэтому: добавить параметр диалога (по образцу существующего `ShowSorting`), например `ShowQuickSearch`, **по умолчанию `false`**. Включать его **только** при вызове из динамического грида и только когда `ПоддерживаетБыстрыйПоиск = true` (QS3). Значение `false` по умолчанию гарантирует, что оба других сценария не затронуты без единой правки на их стороне.

## Что сделать

1. **Чекбокс в строке колонки.** Использовать `<ClayCheckbox>` (controlled: `State` + `OnToggle`) — конвенция проекта, inline `<span>`-глифы 16×16 запрещены. Разместить рядом с существующим переключателем видимости, в общем стиле строки-чипа (`.column-settings-chip`).

2. **Заголовок/подпись** — на русском, коротко: «Поиск» или «Быстрый поиск». Если в диалоге есть шапка со подписями колонок настроек — добавить подпись туда, а не текстом в каждой строке.

3. **Недопустимые по типу колонки** (QS4): чекбокс отображается **заблокированным** (`Disabled`) и снятым, с тултипом на русском, объясняющим причину — например «Поиск недоступен для колонок этого типа». Тултип — через `ClayButton`/`MudTooltip` по существующему в диалоге образцу.
   - Именно `Disabled`, а не скрытие: пользователь должен видеть, что колонка существует и почему по ней нельзя искать.
   - Заблокированный чекбокс невозможно включить ни кликом, ни через клавиатуру.

4. **Изоляция от drag-and-drop.** Порядок колонок в диалоге — jQuery UI Sortable (`clayColumnSettings.js`), область сортировки уже изолирована через JS `closest()` (`.sort-toggle-area`). Новый чекбокс **обязан** быть исключён из захвата перетаскивания тем же приёмом — иначе клик по нему начнёт тащить чип. Проверить руками: клик по чекбоксу не инициирует drag, перетаскивание чипа за свободную область по-прежнему работает.

5. **Применение.** По кнопке «Применить»: набор пересчитывается (QS6), сохраняется в `ClayGridUserParams` (QS5) вместе с остальными настройками диалога, при необходимости данные перезагружаются (QS6).

6. **Отмена.** Существующее поведение — порядок, видимость и сортировка восстанавливаются из snapshot. Флаги быстрого поиска **обязаны попасть в тот же snapshot** и восстанавливаться при отмене. Не заводить отдельный механизм отмены.

7. **Кнопки сброса внизу диалога:**
   - `ClearAll` — сброс **сортировки**. Флаги поиска не трогать.
   - `RestartAlt` — восстановление исходного порядка, видимости и сортировки. Добавить сюда же восстановление флагов быстрого поиска **к админским значениям** из `ColumnsTable` (то есть к состоянию «пользователь ничего не менял»). Зафиксировать это в `///`-комментарии.

8. **Валидация.** Существующая проверка «должна быть видна хотя бы одна колонка» остаётся как есть. **Аналогичной проверки для поиска не добавлять** — ноль колонок поиска это законное состояние (поле просто скрывается, QS6).

## Взаимодействие с видимостью колонки

Скрытая колонка **может** участвовать в быстром поиске — это законно и намеренно: пользователь ищет по полю, которого не видит на экране. Не связывать эти два переключателя и не блокировать один другим.

## Чего НЕ делать

- Не добавлять кнопки «Выбрать все / Снять все» для поиска.
- Не менять раскладку, ширины и стили существующих элементов диалога сверх добавления одной колонки управления.
- Не менять `clayColumnSettings.js` сверх минимально необходимого для изоляции чекбокса от drag-and-drop.
- Не сохранять настройку по клику на чекбокс — только по «Применить» (QS5).

## → verify

1. Динамический грид, колонка `УчаствуетВБыстромПоиске` есть → чекбоксы видны, состояние соответствует итоговому набору из QS6.
2. Динамический грид, колонки нет → чекбоксов нет, диалог работает как раньше.
3. Статический грид → чекбоксов нет, диалог работает как раньше.
4. Печать/Excel → prompt выбора колонок без чекбоксов поиска, `ShowSorting=false`-режим не изменился.
5. Справочная колонка (`КодТипа` в стенде) → чекбокс заблокирован, снят, тултип на русском, включить невозможно.
6. Изменить галки → «Применить» → поле поиска и результаты соответствуют новому набору; настройка в БД (QS5).
7. Изменить галки → «Отмена» → состояние вернулось, в БД ничего не записалось.
8. `RestartAlt` → флаги вернулись к админским значениям; `ClearAll` → флаги поиска не изменились.
9. Drag-and-drop колонок в диалоге по-прежнему работает; клик по новому чекбоксу не тащит чип.
10. `dotnet build` + `dotnet test` зелёные.
