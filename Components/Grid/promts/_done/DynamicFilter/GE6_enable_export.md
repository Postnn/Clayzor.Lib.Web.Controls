# GE6 — включение печати и Excel в динамическом режиме

**Это шаг включения фичи. Если GE1–GE5 не выполнены — ОСТАНОВИСЬ.** До этого момента меню
печати и Excel в динамическом гриде было скрыто (GF15), и весь написанный код лежал мёртвым.
Здесь всё оживает, и здесь же вылезет всё, что было сделано неаккуратно.

Прочитать перед началом (обязательно, до написания кода):

- **`promts/GF15_disable_dynamic_export.md` — что именно там было отключено. Этот шаг его
  откатывает.**
- `Components/Grid/ClayGrid.razor` — меню групповых операций: условия подменю «Печать» и
  «Выгрузка в Excel», условие показа самого меню.
- `Components/Grid/ClayGrid.razor.cs` — `ShowPrint`, `ShowExcel`, `CustomBatchGroups`,
  `_hasBatchOperations` (добавлено в GF15).
- `Kesco.App.Web.Inventory/Components/Pages/Home.razor`.
- `Components/Grid/ClayGrid.ExportMenu.cs` — убедись, что диспетчеры из GE4/GE5 на месте
  во всех ШЕСТИ методах.

## Задача

GF15 скрыл подменю условием `DataLoader is not null`. После GE4/GE5 это условие неверно:
динамический режим печатает и выгружает сам, без `DataLoader`. Возвращаем меню.

## Изменить/создать

**1.** `ClayGrid.razor` — вернуть условия подменю:

```razor
@if (ShowPrint)
{
    @* … подменю «Печать» … *@
}
@if (ShowExcel)
{
    @* … подменю «Выгрузка в Excel» … *@
}
```

То есть ровно как было до GF15: `&& DataLoader is not null` убирается.

**2.** `ClayGrid.razor.cs` — `_hasBatchOperations` (добавлено в GF15):

```csharp
    /// <summary>Есть ли что показать в меню групповых операций.</summary>
    private bool _hasBatchOperations => ShowPrint || ShowExcel || (CustomBatchGroups?.Count > 0);
```

Условия `DataLoader is not null` уходят. Свойство оставь — оно полезно и само по себе:
без него грид рисует пустое меню, когда всё выключено.

**3.** `ClayGrid.razor.cs` — вернуть xml-doc `ShowPrint`/`ShowExcel` к исходному виду: строка
«работает только при заданном `DataLoader` (статический режим)», добавленная в GF15, больше
не верна. Удали её.

**4.** `Home.razor` — вернуть параметры:

```razor
<ClayGrid TEntity="IClayGridRow"
          Dynamic="true"
          Id="clay-grid-root"
          ShowPagination="true"
          ColumnMenuMode="@ColumnMenuMode.Always"
          ShowPrint="true"
          ShowExcel="true" />
```

**5.** Убедись, что в `ClayGrid.ExportMenu.cs` во ВСЕХ шести методах стоит
`if (!Dynamic && DataLoader is null) return;`, а не старое `if (DataLoader is null) return;`.
**Пропустишь один — пункт меню будет виден и молча ничего не сделает, то есть вернётся ровно
тот баг, из-за которого писался GF15.** Проверь список поимённо: `PrintCurrentPageInternal`,
`PrintAllInternal`, `PrintSelectedInternal`, `ExcelCurrentPageInternal`, `ExcelAllInternal`,
`ExcelSelectedInternal`.

**6.** Убедись, что временные правки из блоков «Проверка» GE3/GE4/GE5 удалены: отладочный
вызов `BuildDynamicExportRowsForAll` в `InitDynamicMode`, временные условия `@if (ShowPrint)` /
`@if (ShowExcel)` — они теперь и есть постоянные, а вот отладочный `Console.WriteLine` быть
не должен.

**7.** `GF7_backlog.md` — пункт 2 («Печать и выгрузка в Excel динамического грида») отметить
как реализованный планом GE. Пункт 3 (фильтр по значению) НЕ трогать — он остаётся в бэклоге.

## Не делай

Не убирай `if (!Dynamic && DataLoader is null) return;` из методов экспорта — это защита от
конфигурации «не динамика и загрузчик не передан». Не трогай `CustomBatchGroups` — их
обработчики от `DataLoader` не зависели и не зависят. Не включай `SelectVisible` в `Home.razor`,
если он там не нужен: режим выбора — отдельное решение (GF13). Не трогай статический режим.

## Проверка (ручная, полная — приёмка всего плана GE)

Динамический режим, `?id=140&CLID=9`:

- меню групповых операций видно, в нём подменю «Печать» и «Выгрузка в Excel»;
- «Печать → Текущая страница» / «Все данные» — форма открывается, ячейки заполнены,
  наименования вместо кодов (Тип 5), локальное время (Тип 10);
- «Excel → Текущая страница» / «Все данные» — файл скачивается, числа остаются числами,
  даты датами;
- пункты «Выбранные (N)» появляются только когда что-то выбрано (при `SelectVisible="true"`,
  GF13);
- поставить `ShowPrint="false"` в `Home.razor` → подменю «Печать» исчезло, «Excel» осталось;
  вернуть;
- поставить `ShowPrint="false" ShowExcel="false"` и не задавать `CustomBatchGroups` → меню
  групповых операций не показывается вообще (`_hasBatchOperations`); вернуть;
- **ни один пункт меню не «молчит»**: каждый либо делает работу, либо показывает снекбар
  с ошибкой. Прокликай все шесть.

Статический режим (`MedicalTests.razor`) — приёмка отсутствия регрессии:

- «Печать → Текущая страница / Все данные / Выбранные (N)» — все три работают;
- «Excel → Текущая страница / Все данные / Выбранные (N)» — все три работают;
- с группировкой: групповые строки в печати, Excel Outline в файле;
- имя файла и его содержимое совпадают с эталоном, снятым до GE1.

Отчёт по плану — зафиксируй известные компромиссы, чтобы они не всплыли как баги:

- Тип 10/13 в Excel — текст, а не дата (формат берётся из справочника, GE2);
- Тип 12 в печати и Excel — полный текст, на экране обрезанный (GE2);
- Тип 9 в печати и Excel — тултип, а не картинка (GE2);
- Тип 4 — подпись, гиперссылки в Excel нет (GE2);
- если `GG7` не выполнен — группированный экспорт написан, но недостижим.
