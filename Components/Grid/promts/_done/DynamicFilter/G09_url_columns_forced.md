# G9 — видимость/порядок колонок из URL + исключение forced-условий из сохранения

Прочитать перед началом: G7 (сохранение состояния), G8 (ParsedUrlFilter.IsForced).

Реализовать:
1. Параметр видимости/порядка колонок из URL (`cols+gridId` или `_cols+gridId`, п.6) применять
   по тем же правилам '_'/без '_', что и фильтры (G8): `_` — дефолт (только если нет
   пользовательского), без `_` — принудительно.
2. В G7-сохранении ИСКЛЮЧИТЬ все `IsForced`-условия и принудительные параметры: они применяются
   к запросу, но НЕ попадают в `SaveAsync` (не «залипают» в БД пользователя).

Не делай: не сохраняй forced-значения; не удаляй уже сохранённые пользовательские значения
из-за прихода forced-параметра (forced влияет только на текущий запрос).

Проверка (TG7 частично + ручная):
- URL с `type=in~3,5` (forced) меняет выборку, но после сохранения в п.4 записи `flt140` с этим
  условием НЕТ;
- `_cols140=...` применяется только если пользователь не сохранил свой набор колонок.
