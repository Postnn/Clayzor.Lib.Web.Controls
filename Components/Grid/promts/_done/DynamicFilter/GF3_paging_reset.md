# GF3 — не сбрасывать номер страницы в динамическом режиме

Прочитать перед началом: `Components/Grid/ClayGrid.razor.cs` — параметр `PageNumber`,
`OnParametersSet`; `Components/Grid/ClayGrid.Paging.cs` — `_pageNumber`, `_totalPages`,
`GoToNextPageAsync` и соседи; `Components/Grid/ClayGridPageBase.cs` — кто и зачем передаёт
`PageNumber` в статическом режиме (авто-переход в `ToggleGroup`).

## Дефект

```csharp
protected override void OnParametersSet()
{
    _pageNumber = PageNumber;
    ...
}
```

`PageNumber` — `[Parameter]` со значением по умолчанию `1`. В статическом режиме страница
передаёт его сознательно, чтобы синхронизировать пагинатор при внешних изменениях. В
динамическом режиме пагинация полностью внутренняя: `Home.razor` содержит
`<ClayGrid Dynamic="true" ... />` и `PageNumber` не передаёт — параметр навсегда остаётся `1`.

`OnParametersSet` вызывается при каждом `SetParametersAsync`, то есть при любом ре-рендере
родителя. Каждый такой ре-рендер молча возвращает `_pageNumber` на `1`, при том что данные
в `Items` остаются от текущей страницы. Дефект скрытый: он не проявляется, пока грид пуст
(GF1/GF2), но всплывёт сразу после них.

## Изменить/создать

`ClayGrid.razor.cs`:

```csharp
/// <inheritdoc/>
protected override void OnParametersSet()
{
    // В динамическом режиме пагинация внутренняя: PageNumber снаружи не передают,
    // безусловный сброс убивал переход по страницам.
    if (!Dynamic)
        _pageNumber = PageNumber;

    if (TotalCount > 0 && _pageNumber > _totalPages)
        _pageNumber = _totalPages;
    if (_selectMode)
        _selectAllChecked = ComputeSelectAllState();
}
```

Клампинг `_pageNumber` по `_totalPages` оставить общим для обоих режимов — он защищает от
ситуации «фильтр сузил выборку, текущая страница больше не существует».

`Dynamic` объявлен в `ClayGrid.Dynamic.cs`, это тот же partial-класс — дополнительных
using/полей не нужно.

## Не делай

Не убирай параметр `PageNumber` и не меняй его поведение в статическом режиме. Не добавляй
второе поле состояния страницы. Не трогай `OnInitialized` (`_pageSize = PageSize`) — размер
страницы в динамическом режиме перетирается в `RestoreDynamicState`, и порядок вызовов
(`OnInitialized` → `OnInitializedAsync`) это уже обеспечивает.

## Проверка (ручная)

- `?id=140`, грид с числом страниц > 3: «Вперёд» → «Стр. 2 из K», строки сменились на вторую
  страницу; ещё раз «Вперёд» → «Стр. 3 из K»; «В конец» → последняя страница; «Назад» и
  «В начало» работают симметрично;
- на 3-й странице открыть и закрыть по «Отмена» диалог «Настройка колонок» → номер страницы
  остался 3-м (ре-рендер не сбросил);
- на 3-й странице сменить «Строк на странице» → страница осознанно уходит на 1 (это делает
  `OnPageSizeChangedAsync`, так и задумано);
- на последней странице применить фильтр, сужающий выборку до одной страницы → грид
  показывает «Стр. 1 из 1» и данные, а не пустоту.
