# G10 — Тип 5 (Список): значение из подзапроса, мультивыбор-фильтр

Прочитать перед началом: как реализован фильтр «уникальные значения» (мультивыбор) в гриде
(`ClayColumnValueFilterDialog`, `DistinctValuesResult`), `ColumnTypeDescriptor`.

Создать `ClayListColumnType : ColumnTypeDescriptor` (Тип 5). Формат = SQL (1-я колонка —
значение, 2-я — текст).
1. Загрузка справочника: выполнить SQL из Формат ОДИН раз на загрузку грида ТОЛЬКО через
   `DynamicSql.QueryPairsAsync(Db, formatSql)` (Lib.Entities, G1b), построить словарь
   `value→text`; закэшировать (напр. по тексту SQL) — НЕ выполнять на каждую строку.
2. Вывод ячейки: показывать text по value; если value нет в словаре — показать value как есть.
3. Сортировка в гриде — отключить (клик по заголовку не сортирует). В группировке — сортировать
   по алфавиту text.
4. Фильтр — мультивыбор значений из этого же справочника (как фильтр уникальных значений).
   URL: `key=in~v1,v2`.

Не делай: не выполняй подзапрос в цикле по строкам; не включай сортировку в гриде для этого типа;
НЕ выполняй SQL напрямую из Controls — только через `Clayzor.Lib.Entities` (`DynamicSql`).

Проверка (TG7 + ручная):
- колонка Тип 5 показывает текст; счётчик выполнений подзапроса == 1 при N строках;
- фильтр — мультивыбор; группировка сортирует по тексту; клик по заголовку не сортирует.
