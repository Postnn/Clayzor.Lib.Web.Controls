# A3 — `_opt` внутри грида: единственный источник настроек

**Самый опасный шаг серии.** Одно пропущенное чтение параметра — тихая регрессия: грид
скомпилируется, страница откроется, а какая-нибудь кнопка перестанет появляться. Перед началом
сделай точку откота (git-коммит — по указанию заказчика).

Логика шага: грид получает параметр `Options`, собирает из него (или из legacy-параметров)
поле `_opt` и дальше **читает только `_opt`**. Публичные legacy-параметры остаются на месте
и продолжают работать — потребители не тронуты, регрессия исключена по построению.

## Прочитать

- отчёт **A1** — карта «параметр → файлы и число чтений». Работаешь строго по ней;
- `Components/Grid/ClayGridOptions.cs` (A2);
- `Components/Grid/ClayGrid.razor.cs` — `OnInitialized`, `OnParametersSet`, `NotifyQueryChanged`,
  реализация `IClayGrid`;
- `Components/Grid/ClayGrid.razor` — все чтения параметров в разметке;
- **все** `ClayGrid.*.cs` — по списку из A1.

## 1. Параметр и поле

В `ClayGrid.razor.cs`, в блоке `── Parameters ──`, **первым**:

```csharp
/// <summary>
/// Настройки грида. Если задан — устаревшие параметры тега
/// (<see cref="SelectSql"/>, <see cref="Title"/> и остальные конфигурационные) игнорируются
/// и не должны задаваться одновременно.
/// </summary>
[Parameter] public ClayGridOptions? Options { get; set; }
```

Рядом с полями:

```csharp
/// <summary>
/// Действующие настройки грида — ЕДИНСТВЕННЫЙ источник конфигурации внутри компонента.
/// Собирается в ResolveOptions() из параметра Options либо из устаревших параметров тега.
/// Читать только его; напрямую к миграционным параметрам не обращаться.
/// </summary>
private ClayGridOptions _opt = new();
```

## 2. Сборка

```csharp
/// <summary>
/// Собирает действующие настройки: Options, если задан, иначе — из устаревших параметров тега.
/// При одновременно заданном Options и отличающемся от значения по умолчанию устаревшем
/// параметре бросает исключение: молчаливый приоритет одного источника над другим приводит
/// к настройкам, которых нет в разметке.
/// </summary>
private ClayGridOptions ResolveOptions()
{
    if (Options is null)
        return new ClayGridOptions { /* все MOVE-параметры: Title = Title, … */ };

    var conflicts = CollectLegacyConflicts();
    if (conflicts.Count > 0)
        throw new InvalidOperationException(
            "Одновременно заданы Options и устаревшие параметры тега ClayGrid: " +
            string.Join(", ", conflicts) +
            ". Перенесите их в Options.");

    return Options;
}
```

`CollectLegacyConflicts()` — сравнение каждого legacy-параметра с `ClayGridOptions.Defaults`
(коллекции — по `Count`/`ReferenceEquals`, не поэлементно). Список имён — строками, читаемо
для человека, который увидит исключение в браузере.

Возвращать `Options` **как есть**, без `Clone()`: страница владеет объектом и может менять
его свойства между рендерами — это законный сценарий (например, `Title` из локали). `Clone()`
из A2 пока не вызывается; если по итогам шага он не понадобился — строка в отчёт, удалять его
здесь не надо (он нужен B2/B3 как инструмент страницы).

## 3. Где вызывать — ловушка

Из отчёта A1 известно, что часть параметров читается в `OnInitialized`/`OnInitializedAsync`
(как минимум `PageSize` через `_pageSize = PageSize`, и `Dynamic` при входе в `InitDynamicMode`).
`OnParametersSet` вызывается **после** `OnInitialized`, поэтому сборки только в `OnParametersSet`
недостаточно — на первом рендере `_opt` будет пустым и грид возьмёт `PageSize = 50` вместо
заданного.

Вызывай `_opt = ResolveOptions();` **первой строкой в обоих**: в самом раннем методе
инициализации (по факту кода — `OnInitialized` или `OnInitializedAsync`, проверь) и в
`OnParametersSet`. Два вызова, никакой магии.

**Не** переопределяй `SetParametersAsync` ради «одного правильного места»: приём с
`parameters.SetParameterProperties(this)` + `base.SetParametersAsync(ParameterView.Empty)`
рабочий, но неочевидный, и ломается от любой невнимательной правки. Simplicity First.

## 4. Механическая замена (основной объём)

По карте из A1, файл за файлом: `Title` → `_opt.Title`, `SelectSql` → `_opt.SelectSql` и так
далее для всех **MOVE**-параметров. Включая разметку `ClayGrid.razor` (`_opt` — приватное поле
того же класса, из разметки доступно).

Правила замены:

- **Ничего не оптимизируй по пути.** Не выноси `_opt.SelectSql` в локальную переменную, не
  меняй порядок условий, не сливай похожие выражения. Диф должен читаться как замена
  идентификаторов;
- **STAY-параметры не трогай.** `Items`, `Loading`, `TotalCount`, `PageNumber`, `Columns`,
  `ColumnDefs`, `DataLoader`, `OnAdd`, `OnGroupToggle`, `OnQueryChanged` остаются как есть;
- **осторожно с неуникальными именами.** `Id`, `Title`, `PageSize`, `Dynamic` встречаются в
  чужих контекстах: `_dynamicDef.IdColumn`, `ClayColumnMeta.SqlName`, атрибуты `MudDataGrid`,
  локальные переменные, `DialogParameters`. Замена «найти и заменить по слову» здесь ломает
  код. Каждое попадание — глазами;
- **вычисляемые свойства**, читающие параметры (`HasBatchOperations`, `SelectAvailable`),
  переводятся на `_opt` вместе со всем остальным;
- сами объявления legacy-`[Parameter]` **не удаляются** и не помечаются `[Obsolete]` — это C1.

## 5. Реализация `IClayGrid`

Явные реализации (`string IClayGrid.SelectSql => SelectSql;` и т. д.) в этом шаге переводятся
на `_opt` (`=> _opt.SelectSql`), но **состав интерфейса не меняется** — это B1. То же для
`ColumnMenuMode IClayGrid.ColumnMenuMode`.

## Не делай

- Не меняй публичные сигнатуры: ни одного удаления/переименования/`[Obsolete]` параметра.
- Не меняй `IClayGrid` (B1) и не трогай `ClayGridPageBase`, страницы, диалоги, генераторы.
- Не добавляй в `ResolveOptions` семантических проверок (пустой `SelectSql` и пр.) —
  behavior-preserving.
- Не кешируй `_opt` в статике и не делай его `static`: у страницы может быть два грида.
- Не вводи событие «настройки изменились» и не подписывайся на изменения `Options`.
- Не переводи на `_opt` чтения, которых нет в карте A1, «на всякий случай» — если нашлось
  чтение, не попавшее в карту, это находка: допиши в отчёт и обработай.

  и строка в `CollectLegacyConflicts`. Больше нигде. **Прогони по всем 21 параметру
  поимённо и приложи результат к отчёту**;
