# A1 — инвентаризация параметров: что есть на самом деле

**В этом шаге НЕТ кода.** Ни одной правки файлов. Результат — отчёт в ответе.
Шаг существует потому, что дальше идёт механическая замена ~200 обращений, и ошибка в списке
параметров превратится в тихую регрессию, которую поймает только тестировщик.

## Прочитать

- `Components/Grid/ClayGrid.razor.cs` — блок `── Parameters ──` целиком;
- **все** остальные `Components/Grid/ClayGrid.*.cs` — параметры объявлены не только в основном
  файле (`OnGroupToggle` — в `ClayGrid.Grouping.cs`, `Dynamic`/`DynamicGridId` —
  в `ClayGrid.Dynamic.cs`; проверь, нет ли ещё);
- `Components/Grid/ClayGrid.razor` — какие параметры читаются прямо из разметки;
- `docs/clay-grid.md` — раздел «Параметры» (как источник для сверки, **не** как истину).

## Что сделать

**1. Собери фактический список.** Механически, не по памяти:

```
grep -rn "\[Parameter\]" src/Clayzor.Lib.Web.Controls/Components/Grid/
```

Для каждого параметра выпиши: имя, тип, значение по умолчанию, **файл объявления**.

**2. Классифицируй каждый** по таблице из CGO0 в одну из трёх групп:

- **STAY** — остаётся `[Parameter]` (данные/фрагменты/колбэки);
- **MOVE** — переезжает в `ClayGridOptions`;
- **???** — не подходит ни туда, ни сюда, или в CGO0 не упомянут.

**3. Найди все точки чтения каждого MOVE-параметра.** Для каждого:

```
grep -rn "\bTitle\b" src/Clayzor.Lib.Web.Controls/Components/Grid/
```

Выпиши число попаданий по файлам. Это карта работ для A3 — без неё ты обязательно пропустишь
чтение в каком-нибудь `ClayGrid.Dynamic.Export.cs`. Осторожно с короткими и неуникальными
именами (`Id`, `Title`, `Items`, `Dynamic`, `PageSize`): у них будут попадания в чужие
контексты (`ClayColumnMeta.SqlName`, `_dynamicDef.IdColumn`, `MudDataGrid`-атрибуты,
локальные переменные). Отделяй руками, число «сколько всего» без разбора бесполезно.

**4. Найди внешних потребителей** — кто читает параметры грида снаружи:

```
grep -rn "Grid?\.\|Grid\." src/Clayzor.Lib.Web.Controls/Components/Grid/ClayGridPageBase*.cs
grep -rn "<ClayGrid" src/
```

**5. Проверь особые случаи** и напиши по каждому вывод «да/нет + одна строка почему»:

- есть ли параметр, который грид **записывает** сам (не только читает)? Проверь `Items`,
  `TotalCount` в `ClayGrid.Dynamic.cs`. Такой параметр в options переносить нельзя ни при
  каких условиях — запиши это в отчёт отдельной строкой;
- есть ли параметр, читаемый в `OnInitialized`/`OnInitializedAsync` (то есть **до**
  `OnParametersSet`)? Как минимум `PageSize` (`_pageSize = PageSize`) и, вероятно, `Dynamic`.
  От этого зависит, где в A3 собирать `_opt` — если пропустишь, грид на первом рендере
  получит настройки по умолчанию вместо заданных;
- есть ли параметр, значение по умолчанию которого вычисляется, а не константа
  (`FilterColumnTypes` = новый словарь)? Такие в `ClayGridOptions` инициализируются так же,
  а не как `null`;
- есть ли `[EditorRequired]` / `[CascadingParameter]` среди найденного?
