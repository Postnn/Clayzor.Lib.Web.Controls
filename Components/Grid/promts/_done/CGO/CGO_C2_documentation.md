# C2 — документация

**Это прямое указание заказчика на правку документации** (по `/AGENTS.md` без такого указания
docs и `AGENTS.md` не трогают). Правится только перечисленное ниже; остальную документацию
не касаться.

Причина отдельного шага: после C1 все шаблоны разметки в документации стали неверными.
Разработчик, копирующий их, получит ошибку компиляции — и разбираться будет не с документацией,
а с библиотекой.

## Прочитать

- отчёты **A1** (итоговая классификация) и **C1** (что удалено);
- фактический код после C1: `ClayGridOptions.cs`, блок `[Parameter]` в `ClayGrid.razor.cs`;
- фактическую разметку после B2/B3: `MedicalTests.razor`, `Home.razor` — **шаблоны в
  документации копируются с них, а не пишутся заново**.

### 1. `src/Clayzor.Lib.Web.Controls/docs/clay-grid.md`

- **Раздел «Параметры»** — переписать на два блока:
  - «Параметры тега» — таблица из 10 членов (`Options` + 9 STAY), с колонкой «почему остался
    параметром» для четырёх «живых» (`Items`, `Loading`, `TotalCount`, `PageNumber`) — короткая
    формулировка из CGO0 про новый объект на каждый рендер;
  - «Настройки (`ClayGridOptions`)» — таблица всех свойств: имя, тип, значение по умолчанию,
    описание. Дефолты сверить с кодом, а не переносить из старой таблицы.
- **Раздел «Интеграция на странице»** — заменить пример на фактический из `MedicalTests.razor`,
  включая сборку `_gridOptions` в `OnInitialized`. Добавить абзац-правило: объект настроек
  создаётся один раз в поле страницы; собирать его выражением в разметке нельзя, и почему.
- **Раздел `IClayGrid`** — в таблице членов убрать `SelectSql`/`SearchColumns`/`DefaultOrder`/
  `EditDialogType` (и `ColumnMenuMode`, если он ушёл в B1), добавить `Options`.

- **Блок «Удалённые параметры (больше не используются)»** — он там уже есть, историю с
  `~~OnRowClick~~` и прочими не выбрасывать; дописать новой строкой: конфигурационные параметры
  сведены в `ClayGridOptions` (серия CGO), перечислить их одной строкой через запятую.
- **Раздел «Групповые операции»** — пример `<ClayGrid ... SelectVisible="true" ShowPrint="true"
  ShowExcel="true" />` заменить на вариант через настройки.
