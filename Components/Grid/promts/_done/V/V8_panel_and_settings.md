# V8 (позже). Панель/описание фильтра по значению + тумблер в настройках колонок

Опциональная полировка после рабочего ядра (V1–V7). Можно дробить дальше.

## 8.1 Описание `ValueFilter` для панели, печати и экспорта
Сейчас читаемый текст дерева строит `Filter/ClayFilterDescriptionBuilder.cs`
(`BuildText` / `BuildSegments`), а `ClayGrid.Filtering.cs` вызывает его в
`BuildFilterDescription` / `BuildFilterSegments`. Научить билдер описывать
`ValueFilter`:
- IN:  «{Колонка}: одно из [v1, v2, …]» (+ «, пусто» при `BlankChecked`).
- NOT IN: «{Колонка}: кроме [v1, v2, …]».
- Формат значений — `ColumnTypeDescriptor.Format`; для bool — подписи из меты.
- Сегмент кликабельный → открывает `OpenValueFilterDialog(sqlName)` (маршрут как
  у прочих сегментов панели).
Проверить, что описание попадает в печать/экспорт (`BuildFilterDescription`
используется в `IClayGridDataLoader.BuildPrintHtml*`).

## 8.2 Тумблер режима в диалоге настройки колонок
Дать пользователю включать/выключать фильтр по значению по колонке из
`ClayColumnSettingsDialog` (треб. 1, «в настройках грида»):
- Добавить в `ColumnSettingsItem` поле `AllowValueFilter` (init из меты).
- В диалоге — переключатель на строке колонки (рядом с видимостью/сортировкой).
- Результат сохранять в состояние грида и учитывать в
  `IsValueFilterAvailable` (переопределяет/дополняет дев-флаг `AllowValueFilter`
  из `ClayColumnDef`).
- Персистентность настроек — по той же схеме, что видимость/порядок колонок.

## 8.3 (если нужно) сериализация фильтра по значению
Если состояние фильтра сохраняется в URL/хранилище (см. `_later/12_persistence_extras.md`),
включить `ValueFilter` в сериализацию наравне с `ColumnFilter` (значения
типобезопасно, режим `Negate`, `BlankChecked`).
