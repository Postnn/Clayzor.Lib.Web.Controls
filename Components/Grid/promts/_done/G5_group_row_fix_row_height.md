# G5. Фикс G4: высота строки-заголовка группы схлопывается

G4 решил проблему ширины колонки, но создал новую: `position: absolute` убирает подпись из нормального
потока документа **полностью** — а высота строки в table-раскладке определяется именно нормальным
(не абсолютным) содержимым её ячеек. У строки-заголовка группы после G4 не осталось ни одного элемента
потока, который задавал бы высоту, — строка схлопывается до минимума (видно на скриншоте: строки с
`COVID-19`, `HLA 1 класс` и т.д. заметно ниже строки с реальными данными `279 / РНК коронавируса...`).

C#/Razor-часть не трогаем — проблема только в CSS.

Файл: `app.css`.

## Решение

Не убирать подпись из потока целиком, а обнулить только то, что влияет на ширину колонки. Явно заданная
`width: 0` на обёртке перекрывает то, что «хотят» её переполняющие потомки, при расчёте ширины колонки —
то есть колонка остаётся узкой, ровно как в G4. Но при этом элемент остаётся в нормальном потоке
(`position: static` или `relative`, не `absolute`/`fixed`), поэтому по-прежнему участвует в расчёте
высоты строки — а высота однострочного `white-space: nowrap` содержимого определяется высотой строки
текста/иконки, а не шириной контейнера, так что схлопывания не будет.

Для того, чтобы подпись всё равно визуально рисовалась **поверх** соседних пустых ячеек (а не пряталась
под ними, поскольку они идут позже в DOM-порядке отрисовки), обёртке нужен `position: relative` +
`z-index` — это не убирает её из потока (в отличие от `absolute`), только поднимает её в порядке
отрисовки.

### Правка `app.css`

Заменить блок из G4 (`position: absolute; top:0; bottom:0; left:0;` на `.group-header-cell > div`) на:

```css
/* ── Group Header Rows: полная ширина в интерактивном гриде ──
   width:0 вместо position:absolute (см. G4/G5) — обёртка остаётся в
   нормальном потоке, поэтому по-прежнему задаёт высоту строки, но явная
   width:0 перекрывает intrinsic-ширину переполняющих потомков при расчёте
   ширины колонки. position:relative + z-index поднимают её в порядке
   отрисовки над последующими (пустыми) ячейками той же строки. */

.mud-table-row:has(.group-header-cell) {
    background-color: rgba(5, 22, 77, 0.04) !important;
    border-top: 2px solid var(--lh-navy) !important;
    cursor: pointer;
}

.mud-table-row:has(.group-header-cell):hover {
    background-color: rgba(5, 22, 77, 0.08) !important;
}

.mud-table-row:has(.group-header-cell) > .mud-table-cell {
    background-color: transparent !important;
    border-right: none !important;
}

.mud-table-row:has(.group-header-cell) .group-header-cell {
    overflow: visible;
}

.mud-table-row:has(.group-header-cell) .group-header-cell > div {
    width: 0;
    overflow: visible;
    white-space: nowrap;
    position: relative;
    z-index: 1;
}
```

Обёрнутые в этот `div` кнопка-шеврон и текст — сами по себе не сжимаются («flex-item» по умолчанию не
уменьшается меньше своего содержимого, если явно не задано `min-width: 0`/`flex-shrink`), поэтому при
`width: 0` на родителе они просто визуально переполняют его, не пытаясь сжаться в точку. Если на практике
шеврон или текст всё же начнут сплющиваться — добавить `flex-shrink: 0` на кнопку и на `<p>` с текстом
внутри этого `div`.

## Почему это отличается от G3 и G4

| | В потоке? | Влияет на ширину колонки? | Влияет на высоту строки? |
|---|---|---|---|
| G3 (`overflow:visible`, без `width:0`) | да | да ← баг (колонка растягивается) | да (ок) |
| G4 (`position:absolute`) | нет | нет (ок) | нет ← баг (высота схлопывается) |
| G5 (`width:0`, в потоке) | да | нет (ок) | да (ок) |

## Критерии

- [ ] Высота строки-заголовка группы визуально совпадает с высотой обычной строки детализации.
- [ ] Ширина колонки-хоста по-прежнему не зависит от длины названий групп (регресс из G3 не вернулся).
- [ ] Подпись группы читается целиком на всю ширину, не обрезана и не перекрыта соседними ячейками.
- [ ] Заголовки колонок совпадают по горизонтали со своими данными в строках детализации.
- [ ] Клик по шеврону разворачивает/сворачивает группу как раньше.
- [ ] Многоуровневая группировка — вложенные подписи со своим `padding-left` не съезжают, высота каждого
      уровня одинаковая.
- [ ] Обычные строки данных выглядят как раньше.
- [ ] Ручная проверка в браузере: список с частой группировкой (много подряд идущих групп по 1 шт., как
      на скриншоте), длинные названия групп, узкое окно.
