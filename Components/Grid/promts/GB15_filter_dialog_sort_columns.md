# GB15 — в диалоге настраиваемого фильтра список колонок не отсортирован по алфавиту

Прочитать перед началом: `Components/Grid/ClayGrid.Filtering.cs` — `OpenCompositeFilterDialog`
(построение `filterableCols` из `_columnBySqlName.Values`, передача в диалог через
`x => x.Columns`); `Components/Grid/Filter/ClayFilterExpression.razor` — `MudSelect` с
`@foreach (var col in Columns)` и `<MudSelectItem …>@col.DisplayName</MudSelectItem>`;
`Components/Grid/Filter/ClayFilterDialog.razor.cs` — `[Parameter] Columns`, как он его
прокидывает в группы/выражения; `Components/Grid/Filter/ClayFilterGroup.razor` — проброс
`Columns` дальше.

## Дефект

Выпадающий список колонок в выражении фильтра идёт в порядке `_columnBySqlName.Values`, то есть
в порядке добавления колонок (для динамики — порядок из `Запросы.Колонки`, для статики — порядок
`ClayColumnDef` в разметке). Пользователь ищет колонку по названию — а список не по алфавиту,
приходится глазами перебирать.

`filterableCols` собирается так:

```csharp
var filterableCols = _columnBySqlName.Values
    .Where(c => c.Filterable)
    .ToList();
```

Сортировки по `DisplayName` нет. Правится в одном месте — источнике списка; оба компонента
(`ClayFilterExpression`, и описание в `ClayFilterDialog`) получают уже упорядоченный набор.

## Изменить/создать

`Components/Grid/ClayGrid.Filtering.cs`, `OpenCompositeFilterDialog`:

```csharp
// Отсортировано по отображаемому имени: в выпадающем списке фильтра колонки идут
// по алфавиту, чтобы пользователь находил нужную по названию. Культура — текущая UI
// (StringComparer.CurrentCulture): корректный порядок для кириллицы.
var filterableCols = _columnBySqlName.Values
    .Where(c => c.Filterable)
    .OrderBy(c => c.DisplayName, StringComparer.CurrentCulture)
    .ToList();
```

`StringComparer.CurrentCulture` — чтобы кириллица сортировалась по алфавиту, а не по кодам
символов (как сделал бы `Ordinal`). Приложение русскоязычное, культура потока — русская.

Порядок в выпадающем списке — это только представление в диалоге; на структуру фильтра, на
сохранение (`GridStateSerializer`), на порядок колонок в самом гриде он не влияет. Поэтому
сортируем в точке передачи в диалог, а не в `_columnOrder` и не в `_columnBySqlName`.

Проверь, нет ли второй точки, где тот же диалог получает несортированный список (например,
если `ClayFilterDialog` где-то ещё строит перечень колонок для value-leaf). Если такая точка
есть — примени ту же сортировку там же; если весь список идёт из одного `filterableCols` —
правки в одном месте достаточно.

## Не делай

- Не сортируй `_columnBySqlName` или `_columnOrder` — это порядок колонок В ГРИДЕ, его задаёт
  пользователь (перетаскивание, GB11) и настройка колонок; трогать его нельзя.
- Не сортируй список внутри `ClayFilterExpression.razor` (`@foreach`) — компонент должен
  показывать то, что дали; порядок задаёт поставщик списка. Иначе сортировку придётся дублировать
  в каждом потребителе `Columns`.
- Не используй `OrderBy` без компаратора (по умолчанию для строк берётся текущая культура, но
  явный `StringComparer.CurrentCulture` убирает неоднозначность) и не бери `Ordinal`
  (кириллица уедет не по алфавиту).
- Не меняй порядок операторов, значений или иных выпадающих списков диалога — заявка только
  про список колонок.
- Не трогай диалог value-filter (`ClayColumnValueFilterDialog`) — там список уникальных
  значений одной колонки, не список колонок.

## Проверка (ручная)

- `Kesco.App.Web.Inventory`, `?id=140`: «Настраиваемый фильтр» → в выражении раскрыть список
  колонок → колонки идут по алфавиту по отображаемому имени (кириллица: А, Б, В…);
- добавить второе выражение / вложенную группу → там тот же отсортированный список;
- порядок колонок в самом гриде НЕ изменился (сортировка только в выпадающем списке фильтра);
- применённый ранее фильтр по колонке из середины списка открывается с правильно выбранной
  колонкой (сортировка представления не сбила соответствие `SqlName`↔выбор);
- статический режим (`/medical-tests`): список колонок в фильтре тоже по алфавиту;
- смешанные латиница+кириллица в названиях колонок → порядок разумный (по текущей культуре);
- `dotnet build` + `dotnet test` — зелёные.
