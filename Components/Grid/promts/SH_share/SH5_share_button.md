# SH5 — Кнопка «Поделиться» и диалог создания ссылки

> Требует `SH3`, `SH4`. Правила — в `SH0`.

## 1. Кнопка на панели инструментов

- Компонент — `ClayButton` (конвенция проекта: кнопки с тултипом только так; `ClayButton` сам сбрасывает тултип после клика, что важно, поскольку кнопка открывает диалог).
- Иконка — заливная, в общем ряду с остальными кнопками панели: `Icons.Material.Filled.Share`. Контурные варианты (`*Outline`/`*Outlined`) не использовать — они выбиваются из ряда.
- `Size` и `Color` — как у соседних кнопок панели, без индивидуальных `Style`.
- CSS-класс в общем ряду с `toolbar-*`: `toolbar-share-btn`, стили — в `wwwroot/css/app.css` рядом с соседями, цвета через `--mud-palette-*`.
- Тултип на русском: «Поделиться настройками».
- Кнопка доступна **только в динамическом режиме** грида. В статическом её быть не должно.

## 2. Диалог ввода названия

Новый диалог в `Clayzor.Lib.Web.Controls`. Требования:

- Draggable (конвенция: все модальные окна перетаскиваемые).
- Одно поле ввода: `MudTextField`, `Variant.Outlined`, метка на русском («Название ссылки»), `MaxLength="100"` (соответствует `Название nvarchar(100)`).
- **Значение по умолчанию — название грида** (`ClayGridSettings.Запрос`). Поле при открытии сфокусировано, текст выделен — чтобы можно было сразу набрать своё.
- Кнопки: **«Получить ссылку»** и «Отмена».
- Валидация: `Trim()`, пусто/только пробелы → кнопка «Получить ссылку» заблокирована. Длиннее 100 — невозможно из-за `MaxLength`, но проверку на стороне данных всё равно делает `SH3`.
- `Enter` в поле = «Получить ссылку», `Esc` = «Отмена».
- Один диалог использовать и для создания, и для переименования (`SH7`) — отличаются заголовком, начальным значением и подписью кнопки. Два почти одинаковых диалога не плодить.

## 3. Действие по «Получить ссылку»

Строго в этом порядке:

1. Собрать **текущее состояние грида** в набор пар «имя параметра → значение», используя реестр из `SH4`. Источник — то, что грид показывает сейчас: группировка, фильтры, сортировка, порядок и видимость колонок, быстрый поиск, размер страницы и остальное, что штатно сохраняется в `ClayGridUserParams`.
   - **Не** читать набор из `ClayGridUserParams`: пользователь делится тем, что видит, а часть изменений могла быть ещё не сохранена (решение 7 в `SH0`).
   - Сериализация значений — **та же**, что при сохранении личных параметров. Второго формата быть не должно: иначе `SH8` не сможет их применить.
   - Параметры с пустым значением включать не нужно — набор и так восстановит состояние по умолчанию.
2. Создать запись общей настройки, получить `КодНастройкиОбщей` (`SH3`).
3. Сохранить набор параметров с этим `КодНастройкиОбщей` (`SH3`) — атомарно с шагом 2.
4. Сформировать ссылку (`SH6`).
5. Скопировать в буфер обмена (`SH6`).
6. Показать временное всплывающее сообщение об успешном копировании и **закрыть диалог**.

**Порядок 5 → 6 важен:** сообщение «ссылка скопирована в буфер обмена» показывается только после фактически удавшегося копирования. Если копирование не удалось — поведение по `SH6`, и диалог **не** закрывается молча с ложным сообщением.

## 4. Ловушка: сохранение личных параметров в режиме `sharedId`

Если грид открыт по чужой ссылке, «поделиться» разрешено (решение 6 в `SH0`), но сборка набора на шаге 1 **не должна** попутно сохранить что-либо в личные параметры пользователя. Никаких «сначала сохраним текущее состояние, потом скопируем» — это прямое нарушение требования «в режиме `sharedId` ничего не сохраняется». Запрет обеспечивается choke point из `SH8`; здесь достаточно не обходить его.

## 5. Состояние во время операции

- На время создания ссылки кнопка «Получить ссылку» заблокирована, показывается индикатор (`MudProgressCircular`, `Size.Small`, как при экспорте). Флаг снимается в `try/finally`.
- Повторные нажатия во время операции не должны создавать вторую запись.

## Чего НЕ делать

- Не показывать пользователю саму ссылку в диалоге при успешном копировании — по требованию диалог закрывается. (Показ ссылки — только в сценарии сбоя буфера, `SH6`.)
- Не добавлять галочек «включить фильтры / включить сортировку» — делится всё состояние целиком.
- Не добавлять предпросмотр настроек, счётчик параметров, историю созданных ссылок в этом диалоге (список — отдельный этап `SH7`).
- Не менять существующие кнопки и порядок панели инструментов сверх добавления одной кнопки.

## → verify

1. Кнопка видна в динамическом гриде, отсутствует в статическом, визуально не отличается от соседних, тултип на русском.
2. Диалог открывается перетаскиваемым, поле предзаполнено названием грида, текст выделен.
3. Пустое название → «Получить ссылку» заблокирована. Ввод 100 символов → принимается, 101-й не вводится.
4. Успешный сценарий: в БД появилась запись общей настройки и строки параметров с её `КодНастройкиОбщей`; личные параметры пользователя **не изменились** (сравнить до/после); показано всплывающее сообщение; диалог закрылся.
5. Второй «поделиться» по тому же гриду с тем же состоянием → создалась **вторая** запись, нарушения `UNIQUE` нет (проверка работы `SH2`).
6. Двойной быстрый клик по «Получить ссылку» → создана одна запись.
7. Отмена диалога → в БД ничего не появилось.
8. Узкий экран ≈600px и тёмная тема — кнопка и диалог без дефектов.
9. `dotnet build` + `dotnet test` зелёные.
