# SH7 — Список общих настроек грида

> Требует `SH3`, `SH4`, `SH5`, `SH6`. Правила — в `SH0`.

## Кнопка списка

- Располагается **рядом** с кнопкой «Поделиться» на панели инструментов.
- Показывается **только при наличии общих настроек у текущего грида** — по лёгкой проверке из `SH3` (операция 4), а не по загрузке полного списка.
- Проверка выполняется при загрузке определения грида и **пересчитывается после создания и после удаления** настройки: создал первую ссылку — кнопка появилась без перезагрузки страницы; удалил последнюю — исчезла.
- Компонент — `ClayMenu` (конвенция проекта: выпадающие меню только так, `MudMenu` с `ActivatorContent` запрещён). `ClayMenu` сам строит кнопку-активатор с тултипом и сбрасывает тултип после клика.
- Иконка заливная, в общем ряду с соседями; тултип на русском («Общие настройки»). CSS-класс в ряду `toolbar-*`.

## Загрузка списка

- **При каждом раскрытии список загружается из БД заново** — это прямое требование заказчика. Кэш, «загрузить один раз и держать», обновление только по кнопке — запрещены.
- На время загрузки — индикатор внутри раскрытого списка (`MudProgressCircular`, `Size.Small`). Пустой мигающий список показывать не нужно.
- Список — только свои ссылки текущего пользователя (решение 2 в `SH0`), отсортированные по названию.
- Если к моменту раскрытия настроек не осталось (удалены из другой сессии) — показать короткое сообщение на русском внутри списка и пересчитать видимость кнопки.

## Элемент списка

Название общей настройки + четыре кнопки, все — `ClayButton` с тултипами на русском, иконки заливные, размер как у прочих иконочных кнопок проекта:

| Действие | Иконка (ориентир) | Поведение |
| --- | --- | --- |
| Редактировать | `Icons.Material.Filled.Edit` | Открывает **тот же** диалог, что в `SH5`, с заголовком переименования и текущим названием в поле. По подтверждению — `UPDATE` только поля `Название` (`SH3`). Параметры настройки не трогаются. |
| Удалить | `Icons.Material.Filled.Delete` | **`ConfirmDialog`** с текстом на русском, включающим название удаляемой ссылки. По подтверждению — `DELETE` родительской записи; параметры уходят каскадом (`SH2`). |
| Перейти | `Icons.Material.Filled.OpenInNew` | Открывает соответствующую ссылку в **новом окне** (`SH6`, часть 3). |
| Поделиться | `Icons.Material.Filled.Share` | Копирует соответствующую ссылку в буфер обмена тем же хелпером (`SH6`), со всплывающим сообщением и обработкой сбоя. |

Длинное название (до 100 символов) не должно ломать вёрстку: обрезка по ширине с `text-overflow: ellipsis` и полным названием в `title`/тултипе.

## Закрытие списка

**Любая из четырёх кнопок закрывает список** — прямое требование заказчика. Порядок: сначала закрыть список, затем выполнить действие (открыть диалог, показать подтверждение, открыть окно, скопировать). Иначе модальный диалог откроется поверх раскрытого меню, а `ClayMenu` останется висеть под ним.

Программное закрытие — штатным способом `ClayMenu` (`SH1`, вопрос 15). Не имитировать клик мимо меню через JS.

## Ссылка для элемента списка

Ссылка строится тем же кодом, что в `SH5`/`SH6`, но с `sharedId` **этого** элемента. Хранить готовые ссылки в БД не нужно — они выводятся из текущего URL и идентификатора.

## Ловушки

1. **Список внутри списка.** Диалог редактирования и `ConfirmDialog`, открытые из меню, — модальные и draggable. Проверить, что они не оказываются под меню по `z-index` и что перетаскивание работает.
2. **Обновление после действия.** После переименования и удаления содержимое списка устарело. Специально ничего не «освежать» — список всё равно перезагружается при следующем раскрытии. А вот **видимость кнопки** пересчитать нужно (см. выше).
3. **Удаление настройки, по которой прямо сейчас открыт грид.** Пользователь может удалить ссылку, находясь в режиме `sharedId` этой же настройки. Это допустимо; текущий грид продолжает работать на уже загруженных значениях, но при перезагрузке страницы попадёт в ветку «несоответствие/не найдено» (`SH8`). Специальной обработки не добавлять, поведение отметить в отчёте.

## Чего НЕ делать

- Не добавлять поиск/фильтр/пагинацию по списку настроек.
- Не показывать `КодНастройкиОбщей`, дату, владельца, счётчик параметров — только название и кнопки.
- Не добавлять массовые операции (удалить все) и drag-and-drop сортировку списка.
- Не заводить второй диалог переименования (`SH5` уже сделал общий).
- Не менять существующие пункты и порядок панели инструментов сверх добавления одной кнопки.

## → verify

1. Общих настроек нет → кнопки списка нет.
2. Создать ссылку (`SH5`) → кнопка появилась **без** перезагрузки страницы.
3. Раскрыть список → в БД видно, что запрос выполнен; повторное раскрытие → запрос выполнен **снова** (не кэш).
4. Изменить название в БД руками, раскрыть список → показано новое название (подтверждение перезагрузки).
5. «Редактировать» → список закрылся, диалог открылся с текущим названием, перетаскивается; сохранение меняет только название, параметры настройки в БД целы.
6. «Удалить» → список закрылся, показано подтверждение с названием; отказ ничего не удаляет; подтверждение удаляет запись и её параметры, личные параметры целы; при удалении последней — кнопка списка исчезла.
7. «Перейти» → список закрылся, открылось новое окно с правильным `sharedId`, исходная страница цела.
8. «Поделиться» → список закрылся, ссылка в буфере, всплывающее сообщение показано; при недоступном буфере — поведение из `SH6`.
9. Название из 100 символов не ломает вёрстку списка; полный текст доступен в тултипе.
10. Узкий экран ≈600px, мобильный вид, тёмная тема — без дефектов.
11. `dotnet build` + `dotnet test` зелёные.
