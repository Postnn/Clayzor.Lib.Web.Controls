**Проверка соответствия** (`SH8`):

**Разбор `sharedId`** (`SH8`): вся таблица из п. 1 того этапа.

**Совместимость SQL с 2008 R2:** в сгенерированных запросах нет `OFFSET`, `FETCH`, `MERGE`, `TRY_CONVERT`, `STRING_SPLIT`, `IIF`, `CONCAT_WS`.

Существующие тесты обязаны остаться зелёными **без правок**. Падение существующего теста — регресс, разбираться, а не подгонять ожидание.

## Часть 2. Проверки на уровне БД (стенд из `SH2`)

Выполнить руками в SSMS и зафиксировать результат:

1. Служебная запись `КодНастройкиОбщей = 0` существует; у всех личных параметров это значение.
2. `INSERT` личного параметра (`КодНастройкиОбщей = 0`) с существующим ключом → значение обновилось, дубля нет.
3. `INSERT` того же имени параметра с `КодНастройкиОбщей = N` → отдельная строка, личная не тронута.
4. Повторный `INSERT` по тому же `КодНастройкиОбщей = N` → обновление общей строки, личная не тронута.
5. Один пользователь, одно имя параметра, два разных `КодНастройкиОбщей` → две строки, `UNIQUE` не нарушен.
6. `INSERT` с несуществующим `КодНастройкиОбщей` → падает на внешнем ключе.
7. **`DELETE FROM dbo.ClayGridUserSharedParams WHERE КодНастройкиОбщей = 0` → падает на внешнем ключе, личные параметры целы.** Если запрос прошёл — где-то остался `ON DELETE CASCADE`, немедленно исправить.
8. `DELETE` параметров настройки `N`, затем `DELETE` записи `N` → успешно, личные строки целы.
9. Чтение личных параметров существующим запросом не возвращает строки общих настроек (проверка правки из `SH3`, п. 1a).
10. Новая настройка получает идентификатор ≥ 1.
11. `SELECT * FROM <UserParamsShared>(N)` возвращает параметры настройки `N`; вызов с `0`, с отрицательным и с несуществующим идентификатором даёт пустой набор без ошибки.
12. Объект возвращает строки, созданные другим пользователем — фильтра по пользователю внутри нет.
13. Скрипт схемы применён повторно — без ошибок.

Пункты 3–5 — проверка того, что блокер из `SH0` (ловушка 1) закрыт. Пункты 7 и 9 — проверка того, что не появилось двух новых способов молча потерять пользовательские настройки.

## Часть 3. Ручной сквозной тест

**Создание ссылки**
1. Настроить грид: группировка по колонке, фильтр, сортировка по двум колонкам, изменённый порядок и видимость колонок, быстрый поиск, нестандартный размер страницы.
2. Снять снимок `ClayGridUserParams` для своего пользователя.
3. «Поделиться» → название по умолчанию равно названию грида → «Получить ссылку».
4. Всплывающее сообщение показано, диалог закрылся, ссылка в буфере.
5. Личные параметры **не изменились** относительно снимка.
6. Второй «поделиться» по тому же гриду → вторая запись, ошибок нет.

**Открытие ссылки**
7. Вставить ссылку в новое окно браузера → грид открылся ровно в том состоянии, которое настраивалось в п. 1.
8. Проверить под **другим пользователем** — настройки применились (это проверка отсутствия фильтра по пользователю).
9. Индикатор режима общих настроек виден.
10. Покрутить в этом режиме все настройки, применить диалог колонок, перезагрузить страницу **без** `sharedId` → личные параметры совпадают со снимком п. 2. **Это главная проверка задачи.**
11. Из режима `sharedId` нажать «Поделиться» → новая запись создалась, личные параметры по-прежнему совпадают со снимком.

**Список общих настроек**
12. Кнопка списка появилась после создания первой ссылки, без перезагрузки страницы.
13. Раскрытие списка каждый раз обращается к БД (проверить сменой названия в БД руками).
14. «Редактировать» → список закрылся, название изменилось, параметры настройки целы.
15. «Удалить» → подтверждение с названием; отказ ничего не меняет; подтверждение удаляет запись и параметры; после удаления последней кнопка списка исчезла.
16. «Перейти» → новое окно, правильный `sharedId`, исходная страница цела.
17. «Поделиться» из списка → ссылка в буфере, сообщение показано.

**Негативные сценарии**
18. `sharedId` от другого грида → сообщение об ошибке, **грид не загружен**, запрос данных не выполнялся.
19. `sharedId` несуществующий → как в п. 18.
19a. Смешанный набор: у существующей общей настройки добавить в БД одну строку с посторонним именем параметра → открытие по ссылке даёт сообщение об ошибке, грид **не загружен**, частичного применения настроек нет.
20. `sharedId=abc`, `sharedId=-1`, `sharedId=` (пусто) → как в п. 18.
21. `sharedId=0` и отсутствие параметра → обычный режим, сообщений нет.
22. Кнопка «Открыть без общих настроек» из сообщения → чистый URL, грид на личных настройках, сохранение работает.
23. Удалить настройку, будучи в её режиме `sharedId`, затем перезагрузить страницу → сообщение об ошибке, грид не загружен, без падений.
24. Имитация недоступного буфера обмена (`delete navigator.clipboard` в консоли) → ложного «скопировано» нет, ссылка показана для ручного копирования.

**Регресс и оформление**
25. Статический грид — без изменений; кнопок «Поделиться» и списка нет.
26. Печать, Excel, выделение, группировка, фильтрация, быстрый поиск в обычном режиме — как до задачи.
27. Узкий экран ≈600px, мобильный вид, тёмная тема — панель инструментов, список, диалоги и сообщение об ошибке без дефектов.
