# QS9 — SQL-условие, тесты, ручной тест, отчёт

> Финальный шаг. Требует всех предыдущих. Правила — в `QS0`.

## Часть 1. SQL-условие поиска

Условие строится существующим SQL-билдером (QS7). Здесь — требования, которые должны быть выполнены и **покрыты тестами**.

1. **Форма условия:** `OR` по всем колонкам набора, всё целиком в скобках, приклеивается к остальным условиям через `AND`:
   `... AND (<c1> LIKE @q ESCAPE '\' OR <c2> LIKE @q ESCAPE '\')`

2. **Параметризация.** Значение — Dapper-параметр. Шаблон `%значение%` собирается **в C#**, в параметр передаётся уже готовая строка с процентами. В SQL-текст не подставляется ничего.

3. **Экранирование метасимволов `LIKE`.** Пользовательский ввод может содержать `%`, `_`, `[`. Без экранирования ввод `50%` найдёт всё. Экранировать перед сборкой шаблона и использовать `ESCAPE`. Покрыть тестом.

4. **Приведение типов.** Строковые колонки — напрямую. Нестроковые (числа) — `CAST(<col> AS nvarchar(50))`. `TRY_CONVERT` **недоступен** в 2008 R2.

5. **Даты.** Искать нужно по тому, что пользователь видит. Если в `Формат` колонки задан `dd.MM.yyyy`, то `CAST(дата AS nvarchar)` даст непохожую строку и поиск «не работает» с точки зрения пользователя. Использовать `CONVERT(nvarchar(30), <col>, 104)` (формат `dd.mm.yyyy`).
   **Если в существующем SQL-билдере уже есть конвенция приведения дат — следовать ей, а не вводить вторую.** Решение и причину зафиксировать в отчёте.

6. **Регистронезависимость** обеспечивается collation БД. Не добавлять `LOWER()`/`UPPER()` вокруг колонок — это убивает индексы и не требуется. Проверить фактическое поведение на стенде и отметить в отчёте.

7. **`NULL`-значения** в колонках просто не совпадают — дополнительных `ISNULL` не добавлять.

8. **Обёртка исходного запроса.** Условие применяется к исходному `ClayGridSettings.[SQL]`, обёрнутому по существующей в проекте схеме (алиас `_src`). Не переписывать обёртку, не дублировать её.

## Часть 2. Автотесты

Добавить в `tests/Clayzor.Lib.Web.Controls.Tests` (xUnit). Тесты из предыдущих шагов уже должны быть на месте; здесь — SQL и сводные:

- форма условия: скобки на месте, `OR` внутри, `AND` снаружи;
- одна колонка в наборе → корректное условие без лишних `OR`;
- пустой набор → условие поиска **не добавляется** в SQL вообще (не `AND (1=1)`, не пустые скобки);
- экранирование `%`, `_`, `[`;
- `CAST`/`CONVERT` для нестроковых и дат;
- отсутствие подстановки значения в текст SQL (в сгенерированной строке присутствует имя параметра, а не введённый текст);
- совместимость с 2008 R2: в сгенерированном SQL нет `OFFSET`, `FETCH`, `TRY_CONVERT`, `STRING_SPLIT`, `IIF`, `CONCAT_WS`;
- сводный тест на всю цепочку QS6 (таблица из QS6 — все 8 строк).

Существующие тесты (их 39 на момент постановки задачи) должны остаться зелёными **без правок**. Если какой-то падает — это регресс, разбираться, а не подгонять тест.

## Часть 3. Ручной тест

На стенде из QS2, грид №140:

1. Колонка есть, `Название=1`, `КодИсследования=1`, `КодТипа=1` (справочник), `Активно=0`:
   поле поиска видно; поиск по названию и по коду работает; по типу исследования — нет (колонка недопустима).
2. Ввести `50%` или `_` — поиск ищет подстроку буквально, не «всё».
3. Поиск + фильтр по колонке одновременно → результат соответствует обоим условиям.
4. Поиск + группировка → группы пересчитаны по найденным строкам, счётчики корректны.
5. Поиск + пагинация → `TotalCount` по найденному, страницы листаются.
6. Поиск + режим выбора → выделение сбрасывается при изменении строки поиска.
7. Диалог настройки колонок: снять `Название`, оставить `КодИсследования` → «Применить» → поиск ищет только по коду; в `ClayGridUserParams` одна строка с `Значение='КодИсследования'`.
8. Снять все галки при непустой строке поиска → строка очистилась, поле скрылось, данные полные.
9. F5 и перезапуск приложения → пользовательский выбор сохранился.
10. `UPDATE dbo.ClayGridColumns SET УчаствуетВБыстромПоиске = 0 WHERE КодЗапроса = 140` → поле поиска скрыто.
11. `ALTER TABLE dbo.ClayGridColumns DROP COLUMN УчаствуетВБыстромПоиске` → грид открывается, поля поиска нет, чекбоксов в диалоге нет, **`ClayErrorBar` не появляется**, в логе нет `SqlException`. Затем вернуть колонку скриптом из QS2.
12. Статический грид с поиском → работает как до задачи.
13. Печать и Excel → prompt выбора колонок без изменений.
14. Узкий экран ≈600px, мобильный вид, тёмная тема — без визуальных дефектов.
