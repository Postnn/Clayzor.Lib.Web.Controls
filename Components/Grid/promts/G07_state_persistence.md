# G7 — сохранять/восстанавливать состояние грида на каждой загрузке

Прочитать перед началом: `ClayGrid.Sorting.cs`, `ClayGrid.Grouping.cs`, `ClayGrid.Paging.cs`,
`ClayGrid.Filtering.cs`, `ClayFilterJsonConverter` — какие модели состояния уже есть и как они
сериализуются (особенно фильтр в JSON).

Реализовать в динамическом режиме ClayGrid:
- CLID = query-параметр `opt.ClientIdQueryParam`; если нет — 0.
- Имена параметров: `cols+gridId` (п.6), `flt+gridId` (п.7), `grp+gridId` (п.8), `srt+gridId`
  (п.9), `pgs+gridId` (п.10) — через `BuildParamName`.
- ВОССТАНОВЛЕНИЕ при загрузке: `LoadAsync(clid, gridId, prefixes)` → применить к состоянию:
  - п.6 → видимость+порядок колонок; п.10 → размер страницы; п.9 → сортировка; п.8 →
    группировка; п.7 → фильтр (десериализовать существующим JSON-конвертером).
  - Если сохранённого значения нет — использовать значения по умолчанию из определения (Порядок).
- СОХРАНЕНИЕ: после КАЖДОЙ загрузки данных грида вызвать `SaveAsync` для всех пяти параметров
  текущего состояния (правило 3), с текущим CLID (правило 4).
- Форматы значений: п.7 — JSON композитного фильтра (тот же конвертер); п.6/п.8/п.9 — компактная
  строка «колонка:порядок[:asc/desc]» через запятую (задай единый формат и придерживайся его);
  п.10 — число.

Не делай: не вводи параллельные модели состояния — используй существующие; не сохраняй, если
грид в статическом режиме.

Проверка (ручная + TG6 на сериализацию):
- изменить видимость/порядок/сортировку/группировку/страницу/фильтр → перезагрузить `?id=140&CLID=7`
  → состояние восстановлено; с `?id=140&CLID=9` — своё (пустое/по умолчанию) состояние;
- round-trip сериализации состояния (TG6) не теряет полей.
