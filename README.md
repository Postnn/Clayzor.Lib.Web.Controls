# Clayzor.Lib.Web.Controls

Библиотека web-компонентов **Clayzor** для Blazor — набор переиспользуемых UI-контролов поверх [MudBlazor](https://mudblazor.com/). Ядро библиотеки — feature-rich грид `ClayGrid<T>` с серверной пагинацией, поиском, сортировкой, группировкой и фильтрацией, а также сопутствующие диалоги, редакторы, сервисы и корпоративная тема.

> Проект — часть более крупного решения Clayzor и подключается как Razor Class Library (RCL). Отдельного NuGet-пакета и релизов на данный момент нет.

## Что это

`Clayzor.Lib.Web.Controls` — это Razor Class Library, которая инкапсулирует общие для приложений Clayzor UI-элементы. Библиотека не заменяет MudBlazor, а надстраивается над ним: компоненты возвращают привычную MudBlazor-разметку и стили, но добавляют готовое поведение — серверную работу с данными, единый фильтр, экспорт, корпоративное оформление и удобные обёртки над часто повторяющимися конструкциями.

Главный сценарий — страницы со списками данных: наследуетесь от `ClayGridPageBase<T>`, размещаете на странице `<ClayGrid>` с колонками, и получаете грид с пагинацией, сортировкой, группировкой, составным фильтром, экспортом в Excel и печатью «из коробки».

## Ключевые возможности

- **Серверная работа с данными** — пагинация, поиск, сортировка, группировка и фильтрация выполняются на стороне SQL Server, а не в браузере.
- **Многоуровневая группировка** без ограничения на число уровней, с корректным подсчётом количества элементов и пагинацией по «эффективным» строкам.
- **Составной фильтр** — дерево условий И/ИЛИ с типозависимыми операторами, фильтр по значениям в стиле Excel, сохранение фильтра в URL (query-параметр `filter`).
- **Динамический режим грида** — определение грида (SQL, колонки, кнопки) читается из БД; 13 типов колонок (число, текст, дата, ссылка, список, иконка, HTML, локальное время и др.).
- **Экспорт в Excel** (через ClosedXML) и **печать** — текущая страница, все данные или только выбранные строки, с сохранением группировки.
- **Готовые диалоги** — редактирование записи, подтверждение, настройка колонок, настройка фильтров.
- **Корпоративная тема** ClayTheme (DarkNavy + золотой акцент) с единым источником фирменных цветов.
- **Обёртки-хелперы** — `ClayButton`, `ClayMenu`, `ClayCheckbox`, `ClayComboBox`, `ClayErrorBar` — убирают повторяющуюся разметку MudBlazor.

## Технологии и зависимости

- **.NET 10** (`net10.0`), `Microsoft.NET.Sdk.Razor`, включены `ImplicitUsings` и `Nullable`.
- **MudBlazor** `9.*` и **MudBlazor.Extensions** `9.4.1`.
- **ClosedXML** `0.105.0` — генерация Excel.
- Ссылки на соседние проекты решения: `Clayzor.Lib.Entities` (сущности, доступ к данным) и `Clayzor.Lib.Web.Settings` (настройки).
- Клиентские хелперы на JavaScript в `wwwroot` (часовой пояс клиента, сохранение прокрутки, drag-and-drop настройки колонок).

## Подключение

Библиотека рассчитана на использование внутри решения Clayzor через `ProjectReference`.

**1. Добавьте ссылку на проект:**

```xml
<ProjectReference Include="..\Clayzor.Lib.Web.Controls\Clayzor.Lib.Web.Controls.csproj" />
```

**2. Зарегистрируйте сервисы** (`Program.cs`). Помимо стандартной регистрации MudBlazor, потребуются сервисы библиотеки — например, обработчик ошибок SQL (`ClayErrorService` / `ISqlErrorHandler`) и, при использовании динамического грида, опции:

```csharp
builder.Services.AddMudServices();

// Обработчик ошибок БД для ClayErrorBar
builder.Services.AddScoped<ClayErrorService>();

// Только если используется динамический режим грида
builder.Services.AddClayGridDynamic(); // читает секцию "ClayGrid:Dynamic" из конфигурации
```

**3. Подключите стили RCL** в корневом документе (`App.razor` / `index.html`) вместе со стилями MudBlazor:

```html
<link href="_content/Clayzor.Lib.Web.Controls/css/clay.css" rel="stylesheet" />
```

**4. Примените тему** в `MainLayout`:

```razor
<MudThemeProvider Theme="ClayTheme.Theme" />
```

> Точный состав регистрации сервисов и разметки может отличаться в зависимости от вашего приложения — сверяйтесь с `AGENTS.md` и документацией в `docs/`.

## Быстрый старт с ClayGrid

Страница наследуется от `ClayGridPageBase<T>`, а конфигурация SQL передаётся параметрами `<ClayGrid>`:

```razor
<ClayGrid TEntity="IClayGridRow"
          DataLoader="this"
          SelectSql="@SQLQueries.SELECT_МоиЗаписи"
          SearchColumns="@(new[] { "НазваниеАнализа", "TestTypeName" })"
          DefaultOrder="Порядок, НазваниеАнализа"
          FilterColumnTypes="@FilterColumnTypes"
          OnGroupToggle="ToggleGroup">

    <ClayColumnDef ColumnId="1" SqlName="Код" DisplayName="Код" Groupable="true" Filterable="true" />
    <ClayColumnDef ColumnId="2" SqlName="НазваниеАнализа" DisplayName="Название" Filterable="true" />

    <ClayColumn TEntity="IClayGridRow" ColumnId="2">
        <CellTemplate>
            @if (context.Item is DetailRow<MyEntity> detail)
            {
                <MudText>@detail.Item.Name</MudText>
            }
        </CellTemplate>
    </ClayColumn>
</ClayGrid>
```

`ClayGridPageBase<T>` сам читает `SelectSql`, `SearchColumns`, `DefaultOrder`, строит `WHERE` и вызывает загрузку данных. Заголовки групп (`ClayGroupHeader`) грид рендерит автоматически — вручную вставлять их в шаблон не нужно, страница только подписывается через `OnGroupToggle`.

## Основные компоненты

| Компонент | Назначение |
| --- | --- |
| `ClayGrid<T>` | Грид с серверной пагинацией, поиском, сортировкой, группировкой и фильтрацией. При заданном `EditDialogType` добавляет сервисную колонку редактирования. |
| `ClayGridPageBase<T>` | Базовый класс страниц с гридом: `LoadData`, `ToggleGroup`, `OpenAddDialog`, авто-вывод типов колонок. |
| `ClayColumn<T>` | Колонка грида с авто-заголовком; берёт метаданные из `ClayColumnDef` по `ColumnId`. |
| `ClayColumnDef` | Невидимый регистратор метаданных колонки (SQL-имя, отображаемое имя, группируемость, фильтруемость и т. д.). |
| `ClayGroupHeader` | Заголовок строки группы (иконка сворачивания + счётчик), рендерится гридом автоматически. |
| `ClayEditForm<T>` | Диалог редактирования с валидацией, сохранением и удалением. |
| `ClayComboBox<TItem>` | Выпадающий список для справочников (`ILookupEntity`). |
| `ClayErrorBar` | Баннер ошибок БД с детализацией (SQL, параметры). |
| `ConfirmDialog` | Диалог подтверждения. |
| `ClayColumnFilterDialog`, `ClayColumnValueFilterDialog`, `ClayFilterDialog` | Диалоги настройки условий фильтра, фильтра по значениям (Excel-style) и составного фильтра. |
| `ClayColumnSettingsDialog` | Настройка порядка, видимости, сортировки и фильтра колонок (drag-and-drop). |
| `ClayButton`, `ClayMenu`, `ClayCheckbox` | Обёртки над MudBlazor-конструкциями (кнопка с тултипом, меню-активатор, tri-state чекбокс). |
| `ClayTheme`, `ClayColors` | Корпоративная тема и единый источник фирменных цветов. |

Ключевые интерфейсы: `IClayGrid` (контракт грида), `IClayGridDataLoader` (обратные вызовы загрузки/экспорта/печати), `ILookupEntity` (справочная сущность), `ISqlErrorHandler` (обработка ошибок SQL).

## Серверная архитектура грида

Группировка и фильтрация выполняются на SQL Server, а не в браузере (подход, близкий к DevExpress/DevExtreme Grid).

- **Группировка** — два запроса: агрегатный `GROUP BY` + `COUNT(*)` и запрос детальных строк с `ROW_NUMBER()`. Родительские узлы для многоуровневой группировки строятся в C#, число уровней не ограничено. Реализация — `ClayGroupingEngine`.
- **Фильтрация** — единое дерево `ClayFilterGroupNode` (узлы И/ИЛИ + листья `ColumnFilter` и `ValueFilter`). SQL собирает `ClayCompositeSqlBuilder.Build`: имена колонок берутся только из белого списка, значения — только через параметры Dapper.
- **Типы колонок** описываются дескрипторами (`ColumnTypeDescriptor` и наследники в `Components/Grid/ColumnTypes/`) — единая точка операторов, парсинга и форматирования.
- **Пагинация** считается по «эффективным» строкам: каждый заголовок группы и каждая строка детализации — одна строка; при сворачивании/разворачивании групп страница пересчитывается.

## Динамический грид

Пакет `Components/Grid/Dynamic/` позволяет строить грид по определению, хранящемуся в БД: SQL-запрос, набор колонок и кнопки действий читаются во время выполнения. Поддерживается 13 типов колонок (`ClayColumnKind`): число, текст, дата, ссылка, список-справочник, иконка, булево, HTML (с санитайзингом), локальные дата/время из UTC и др. Настройки задаются через `ClayGridDynamicOptions` (секция `"ClayGrid:Dynamic"`), состояние грида и пользовательские параметры сохраняются в БД.

## Экспорт и печать

- **Excel** — через ClosedXML; экспорт текущей страницы, всех данных или выбранных строк, с сохранением группировки (интерливинг заголовков групп в C#).
- **Печать** — генерация HTML с теми же MudBlazor-классами, что и на экране, и встроенным `@media print` CSS; печать «как на странице», всех данных или выбранных.
- Чтение значений ячеек абстрагировано через `IClayGridCellReader` (рефлексия для статического режима, словарь — для динамического).

## Тема и стили

- `ClayTheme` — корпоративная тема (тёмно-синий + золотой акцент). Значения палитры берутся из `ClayColors` (единый источник фирменных hex-цветов) и должны быть C#-литералами, а не `var(...)`, поскольку MudBlazor парсит их через `MudColor.Parse()`.
- Общие стили грида, деревьев, чипов и диалогов живут в `wwwroot/css/clay.css` и подключаются через `_content/Clayzor.Lib.Web.Controls/css/clay.css`. Правьте именно этот файл, а не копии в приложениях (см. `STYLE_RULES.md`).

## Разработка

- **`AGENTS.md`** — основной ориентир для разработчиков и AI-агентов: обзор компонентов, codebehind-структура (`ClayGrid` и `ClayGridPageBase` разнесены по тематическим partial-файлам), правила модификации и история выполненных задач.
- **Codebehind-структура.** Новые поля и методы добавляйте в соответствующий тематический partial-файл (например `ClayGrid.Filtering.cs`, `ClayGrid.Grouping.cs`), а не в основной `*.razor.cs`. Базовый класс и интерфейсы объявляются только в основном файле.
- **Правила стиля.** Соблюдайте `STYLE_RULES.md`; при наличии в решении файла `build/StyleGuard.targets` он подключается автоматически и проверяет соглашения.
- **Группировка.** Запрещено использовать встроенную группировку MudBlazor (`GroupBy`/`Groupable`/`GroupTemplate`) и задавать `SortMode` — порядок и группы определяет серверный SQL.

## Заметки об архитектуре

Несколько наблюдений, которые помогают быстрее сориентироваться в коде:

- Библиотека последовательно выносит логику работы с данными на сервер и держит единый источник истины для фильтра (дерево `ClayFilterGroupNode`), что делает поведение статического и динамического режимов согласованным.
- Крупные компоненты (`ClayGrid`, `ClayGridPageBase`) намеренно разбиты на partial-файлы по темам — это упрощает навигацию, но означает, что «одну фичу» стоит искать не в одном файле, а в паре тематических.
- Много внимания уделено безопасности SQL (белые списки колонок, параметры) и повторному использованию (общие мапперы/ридеры для статики и динамики, единые строковые константы UI).
- Это внутренняя корпоративная библиотека: она опирается на соседние проекты решения (`Clayzor.Lib.Entities`, `Clayzor.Lib.Web.Settings`) и на собственный слой доступа к данным, поэтому вне решения «как есть» не соберётся.
